# 代码审查技能

当用户要求审查代码、检查代码质量、或查看变更时使用。

## 审查流程

1. 用 `ssh_execute` 执行 `git diff HEAD~1` 查看最近变更
2. 或用 `ssh_read_file` 读取指定文件
3. 逐项检查以下维度：

## 检查维度

### 安全性（最高优先级）
- SQL注入风险
- XSS风险
- 敏感信息泄露（API Key、密码硬编码）
- 未验证的用户输入
- 不安全的文件操作路径

### 正确性
- 逻辑错误
- 边界条件处理
- 错误处理完整性
- 类型安全

### 性能
- 内存泄漏风险
- 不必要的重复计算
- 大量数据的处理方式
- 数据库查询优化

### 可维护性
- 代码可读性
- 函数职责单一性
- 注释充分性
- 命名规范

## 输出格式

对每个问题标注严重程度：
- 🔴 严重：必须修复（安全漏洞、逻辑错误）
- 🟡 警告：建议修复（性能问题、可维护性）
- 🔵 建议：可选改进（代码风格、命名）
