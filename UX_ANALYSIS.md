# AI 编程平台 - 用户体验分析报告

## 一、核心架构概览

项目基于 Next.js 16 + React 19 + Tailwind CSS 4，包含三大核心模块：
- 对话聊天页（首页）
- 编程工作区（workspace）
- 图片生成（image-gen）

后台管理含 10 个 TAB，功能覆盖面广。

## 二、用户体验问题清单（按严重程度排序）

### P0 - 阻塞性问题

1. **组件颗粒度爆炸**
   - ChatInterface.tsx 单文件超 800 行，集成了对话管理、文件上传、拖拽、SSE 流式渲染、编辑重发、定时任务、图片生成等十几个功能
   - 工作区 AiChat.tsx 同样超 500 行，与 ChatInterface 存在大量重复逻辑
   - 影响：难以维护，任何修改都可能引入回归 bug

2. **主题系统不完整**
   - package.json 中安装了 `next-themes`，但 layout.tsx 未使用 ThemeProvider
   - globals.css 只声明了深色主题变量，没有浅色主题
   - 用户无法切换主题，所有页面锁定深色模式
   - 影响：与 DESIGN.md 中"浅色模式：纯白基底 #ffffff"的设计文档不一致

3. **错误处理原始**
   - 大量使用 `alert()` 弹窗提示错误（admin 页面尤甚）
   - API 调用失败时很多地方只 `console.error`，用户无感知
   - 网络异常时可能白屏或卡住
   - 影响：生产环境体验低劣

### P1 - 体验问题

4. **消息气泡可读性问题**
   - 用户消息使用纯色 `bg-primary text-primary-foreground`，长文本在紫色背景上阅读疲劳度高
   - AI 消息使用 card 背景 + border，与页面背景对比度不足
   - 代码块背景 `#0d1117` 与整体深色主题不协调

5. **工作区布局僵硬**
   - 三个面板（文件树/编辑器/AI聊天）使用固定百分比宽度
   - 虽然安装了 `react-resizable-panels`，但未使用，而是用简陋的 border hover 模拟 resize
   - 终端面板需要手动点击显示/隐藏，缺乏快捷键

6. **导航体验生硬**
   - 首页/工作区/图片生成之间跳转是整页刷新，无过渡动画
   - 顶部导航栏三个页面风格不统一（首页是按钮组，工作区只有返回链接）
   - 缺少面包屑导航，用户容易迷失当前位置

7. **对话历史管理弱**
   - 侧边栏搜索只能匹配标题，不支持全文搜索
   - 对话列表只显示标题和日期，缺少预览或消息数
   - 删除对话无二次确认（只有停止传播）

8. **拖拽上传交互可优化**
   - 拖拽时半透明遮罩覆盖整个聊天区域，遮挡了已有消息
   - 附件预览区域在输入框上方，空间占用大

### P2 - 细节问题

9. **代码块复制按钮文案为英文**
   - `Copy` / `Copied` 未做中文国际化

10. **空状态/加载态不完整**
    - 部分列表缺少 skeleton 加载态
    - 网络请求失败时无重试按钮

11. **定时任务功能鸡肋**
    - 创建任务的 UI 在模态框中，缺少任务状态展示
    - 无任务执行日志/结果查看

12. **移动端适配不完整**
    - 首页侧边栏在移动端有全屏遮罩，工作区面板无移动适配
    - 输入框在大屏和小屏上的 padding 不一致

## 三、优化方案

### 阶段一：架构重构（3天）

#### 1.1 拆分 ChatInterface（核心）
将 ChatInterface.tsx 拆分为以下独立组件：

```
components/chat/
├── ChatInterface.tsx          # 主容器，组合子组件
├── use-chat.ts                # 对话状态管理 Hook
├── use-conversations.ts       # 对话列表管理 Hook
├── use-streaming.ts           # SSE 流式渲染 Hook
├── use-file-upload.ts         # 文件上传/拖拽 Hook
├── ConversationSidebar.tsx    # 对话历史侧边栏
├── ChatMessages.tsx           # 消息列表 + 自动滚动
├── ChatInput.tsx              # 输入框 + 附件预览
├── ChatToolbar.tsx            # 顶部工具栏（模型选择/模式切换）
├── ThinkingIndicator.tsx      # 思考动画
├── TaskPanel.tsx              # 定时任务面板（独立抽出）
└── message-actions.tsx        # 编辑/重发/导出操作按钮
```

#### 1.2 工作区 AiChat 复用 Chat 组件
工作区的 AiChat.tsx 应该复用上述组件，而非重新实现一套。

#### 1.3 统一状态管理
创建全局 ChatProvider，管理：
- 当前对话 ID
- 消息列表
- 模型选择
- 对话模式

### 阶段二：体验升级（3天）

#### 2.1 完成主题系统
在 layout.tsx 中集成 next-themes 的 ThemeProvider：
- 声明完整的浅色/深色主题变量
- 添加主题切换按钮（在导航栏右侧）
- 遵循 DESIGN.md 的配色规范

#### 2.2 全面替换 alert 为 Toast
使用已安装的 sonner 库：
- 成功提示：`toast.success('操作成功')`
- 错误提示：`toast.error('失败原因')`
- 加载提示：`toast.loading('处理中...')`

#### 2.3 消息气泡样式优化
- 用户消息：使用半透明渐变背景而非纯色填充
- AI 消息：增加左侧视觉指示线，提升可读性
- 代码块：统一使用与整体主题一致的深色背景

#### 2.4 工作区面板可拖拽
用 `react-resizable-panels` 替换自定义 border resize：
- 支持面板最小/最大宽度限制
- 拖拽手柄视觉优化
- 保存面板布局到 localStorage

### 阶段三：功能增强（2天）

#### 3.1 对话全文搜索
在对话侧边栏中：
- 点击搜索时请求后端全文搜索 API
- 搜索结果高亮关键词
- 增加搜索筛选条件（日期范围、模型）

#### 3.2 快捷键系统
- `Cmd/Ctrl + K`：全局命令面板
- `Cmd/Ctrl + Enter`：发送消息
- `Cmd/Ctrl + Shift + T`：切换终端
- `Cmd/Ctrl + N`：新建对话
- `/` 快捷键提示浮层

#### 3.3 图片生成体验优化
- 添加拖拽上传参考图功能
- 生成进度条（非 spinning）
- 图片对比模式（原图 vs 生成图）
- 右键菜单（复制/下载/设为参考）

#### 3.4 定时任务增强
- 任务执行日志可查看
- 循环任务（每天/每周）
- 任务完成后发送通知
- 任务状态标签（待执行/执行中/已完成/失败）

### 阶段四：细节打磨（2天）

#### 4.1 国际化
- 代码块复制按钮文案改为中文
- 所有用户可见文案统一中文化

#### 4.2 骨架屏
为以下页面添加 skeleton 加载态：
- 项目列表
- 对话历史
- 图片历史
- 文件树

#### 4.3 过渡动画
- 页面切换添加淡入淡出
- 消息出现动画微调（当前只有 fadeIn，可加入 slide）
- 侧边栏展开收起的弹性动画

#### 4.4 移动端适配
- 工作区：移动端自动切换为上下布局（文件树在上，编辑器居中，聊天在下）
- 图片生成：控制面板改为底部抽屉
- 全局：safe-area-inset 统一处理

## 四、衡量指标

| 指标 | 当前 | 目标 |
|------|------|------|
| Lighthouse Performance | 未测 | 85+ |
| First Contentful Paint | 未测 | < 1.5s |
| 组件文件平均行数 | ~300 | < 150 |
| 用户操作反馈时间 | 无反馈 | < 200ms |
| 错误提示覆盖率 | ~30% | 100% |
| 移动端可用功能 | ~60% | 100% |
| 主题模式 | 仅深色 | 深色 + 浅色 |

## 五、不建议做的

1. 不要引入大型状态管理库（Redux/Zustand），React Context + Hook 已足够
2. 不要重写后端 API 接口，前端优化应兼容现有接口
3. 不要过度动效，保持 DESIGN.md 中"柔和、沉浸"的气质
4. 不要添加第三方登录/用户系统，目前的管理员登录模式够用
5. 不要盲目追求 SSR，当前 SPA 模式对 AI 对话场景更合适
